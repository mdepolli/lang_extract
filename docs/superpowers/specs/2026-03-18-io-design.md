# LangExtract I/O — Design Spec

## Goal

Add serialization and deserialization of extraction results to/from plain maps and JSONL files. This enables storage, debugging, and interop with external systems without coupling consumers to LangExtract's struct types.

## Modules

### `LangExtract.IO`

**Serialize — `to_map/2`:**

```elixir
@spec to_map(String.t(), [Span.t()]) :: map()
LangExtract.IO.to_map(source, spans)
```

Returns a plain map:

```elixir
%{
  "text" => "Romeo and Juliet is a tragedy...",
  "extractions" => [
    %{
      "class" => "work",
      "text" => "Romeo and Juliet",
      "byte_start" => 0,
      "byte_end" => 16,
      "status" => "exact",
      "attributes" => %{"type" => "tragedy"}
    }
  ]
}
```

Each span is converted to a plain map with string keys. The `status` atom is converted to a string (`"exact"`, `"fuzzy"`, `"not_found"`). `byte_start` and `byte_end` are `nil` for `:not_found` spans (serialized as `null` in JSON).

**Deserialize — `from_map/1`:**

```elixir
@spec from_map(map()) :: {:ok, {String.t(), [Span.t()]}} | {:error, :invalid_data}
LangExtract.IO.from_map(map)
```

Converts a plain map back to `{source_text, [%Span{}]}`. Parses the `"status"` string back to an atom. Returns `{:error, :invalid_data}` if the map is missing required keys.

**JSONL — `save_jsonl/2` and `load_jsonl/1`:**

```elixir
@spec save_jsonl([{String.t(), [Span.t()]}], Path.t()) :: :ok | {:error, term()}
LangExtract.IO.save_jsonl(results, path)

@spec load_jsonl(Path.t()) :: {:ok, [{String.t(), [Span.t()]}]} | {:error, term()}
LangExtract.IO.load_jsonl(path)
```

`save_jsonl/2` takes a list of `{source, spans}` tuples, converts each to a map via `to_map/2`, and writes one JSON object per line.

`load_jsonl/1` reads a JSONL file, parses each line via `from_map/1`, and returns the list of results. Skips empty lines. Returns `{:error, :invalid_data}` if any line fails to parse.

## Edge Cases to Test

### to_map/2
- Spans with all statuses (`:exact`, `:fuzzy`, `:not_found`)
- `:not_found` span has `nil` byte_start/byte_end
- Attributes preserved as-is (including nested maps)
- Empty spans list → empty extractions array
- Class and attributes fields present on span

### from_map/1
- Valid map round-trips: `to_map |> from_map` returns equivalent data
- Status strings parsed back to atoms
- `nil` byte offsets preserved
- Missing `"text"` key → `{:error, :invalid_data}`
- Missing `"extractions"` key → `{:error, :invalid_data}`

### save_jsonl/2 + load_jsonl/1
- Round-trip: save then load returns same data
- Multiple results saved as separate lines
- Empty results list → empty file
- load_jsonl on nonexistent file → `{:error, reason}`

## Out of Scope

- URL fetching (use HTTPower directly)
- CSV dataset loading (consumers can handle their own input formats)
- Progress bars
- Streaming/lazy loading (load entire file into memory)
